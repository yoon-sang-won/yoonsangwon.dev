# AI 글쓰기 운영 흐름

AI는 사용자의 실제 메모를 정리하는 보조 도구다. 경험의 출처, 사실의 책임과 공개 결정은 항상 사람에게 있다.

## 메모를 초안으로 바꾸기

1. 먼저 직접 겪은 문제, 시도, 결과와 아직 모르는 점을 짧게 메모한다.
2. 회사명, 사람 이름, 계정 정보, 내부 URL과 비밀값을 제거하거나 일반적인 표현으로 바꾼다.
3. AI에게 글의 종류(Writing, Notes, Lab), 예상 독자와 유지해야 할 사실을 알려 준다.
4. AI에게 빠진 사실을 지어내지 말고 불확실한 곳을 질문이나 표시로 남기라고 요청한다.
5. 받은 초안을 원래 메모와 대조해 실제로 하지 않은 경험, 과장과 반복을 제거한다.
6. 필요한 공식 문서나 원자료를 사람이 다시 확인한다.
7. Pages CMS에 `draft: true`로 저장하고 공개 전 체크리스트를 거친다.

## AI에게 제공해도 되는 정보

- 이미 공개된 공식 문서와 공개 URL
- 공개 저장소의 코드 중 공개해도 되는 부분
- 개인이 직접 작성한 일반적인 학습 메모
- 재현을 위해 필요한 공개 버전, 브라우저와 운영체제 정보
- 개인·회사·고객을 식별하지 않도록 바꾼 문제 상황

공개 정보라도 저작권이 있는 글 전체를 그대로 제공하거나 재작성하게 하지 않는다. 필요한 부분만 요약하고 출처 링크를 남긴다.

## 제공하면 안 되는 정보

- API key, token, password, cookie와 인증 헤더
- 비공개 저장소 코드와 회사 내부 문서
- 고객·동료·가족의 개인정보와 대화 내용
- 내부 URL, 서버 주소, 계정명, 보안 설정과 장애 로그 원문
- 공개 전 제품 계획, 계약, 비용, 매출과 성과 자료
- 라이선스나 공개 권한을 확인하지 않은 이미지와 문서

가림 처리를 했더라도 문맥만으로 조직이나 사람을 알아볼 수 있는지 다시 확인한다.

## AI가 맡을 수 있는 작업

- 메모를 Writing, Notes, Lab 초안 구조로 정리하기
- 제목과 description 후보 만들기
- 중복 문장, 긴 문장과 모호한 표현 찾기
- 문단 순서와 소제목 제안하기
- 공개된 근거에서 추가 확인이 필요한 주장 목록 만들기
- Markdown 문법, 링크 문구와 이미지 alt 초안 점검하기
- 발행 체크리스트에 맞춘 누락 항목 검토하기

AI가 제안한 링크, 버전, 인용, 명령어와 코드가 실제로 존재하고 동작한다고 가정하지 않는다.

## 사람이 반드시 확인할 부분

- 경험, 시도와 결과가 실제로 있었는지
- 사실, 날짜, 버전, 코드와 링크가 정확한지
- 실패 원인과 결론이 증거보다 과장되지 않았는지
- 글이 자신의 판단과 말투를 유지하는지
- 회사·고객·개인의 비공개 정보가 남지 않았는지
- 인용과 이미지의 사용 권한이 있는지
- 제목, description, 날짜와 분류가 맞는지
- 모바일 화면과 실제 공개 페이지가 정상인지

## 가상의 경험과 성과 금지

AI가 빈 부분을 자연스럽게 채웠더라도 직접 하지 않은 작업, 사용하지 않은 도구, 존재하지 않는 프로젝트, 수치, 기간, 반응과 성과는 모두 삭제한다. 예시가 필요하면 “가상 예시”라고 명확히 표시하고 실제 경험처럼 서술하지 않는다.

## 회사 정보 노출 검수

공개 전 원문 메모, AI 입력, 최종 본문, 코드 블록과 이미지까지 모두 확인한다. 회사명을 지웠더라도 화면 구조, 도메인, 저장소명, 티켓 번호, 파일 경로와 고유한 장애 상황이 조직을 식별할 수 있다. 공개 가능 여부가 불확실하면 구체 정보를 더 일반화하거나 해당 부분을 게시하지 않는다.

## Pages CMS에 초안 등록

1. [콘텐츠 편집 원칙](editorial-guide.md)에 따라 Writing, Notes 또는 Lab을 고른다.
2. 영문 소문자와 하이픈으로 공개 URL용 파일명을 확정한다.
3. 제목, description과 게시 예정일을 입력한다.
4. `draft`가 켜진 상태에서 본문을 붙여 넣는다.
5. Source 모드에서 Markdown, 코드 블록, 링크와 이미지 alt를 확인한다.
6. 저장해 GitHub commit을 만든다. 저장하지 않은 브라우저 입력은 초안이 아니다.

여러 기기에서 이어 쓸 때는 [초안 작성 흐름](pages-cms-draft-workflow.md)을 따른다.

## 공개 전 최종 검토

AI가 만든 초안을 바로 공개하지 않는다. 최소 한 번은 원본 메모를 보지 않고 글만 읽어 논리와 반복을 확인하고, 다시 원본 메모와 대조해 사실을 확인한다. [발행 체크리스트](publishing-checklist.md)를 모두 통과한 뒤에만 `draft`를 해제한다. 불확실한 항목이 남으면 초안으로 유지한다.
