# 첫 Writing 글 검토 기록

대상: `src/content/writing/why-this-blog-is-text-first.md`

상태: `draft: true`. production의 목록, 상세 route, Archive, RSS와 sitemap에는 포함되지 않는다.

## 사용자에게서 제공받아 사용한 사실

- 개인 도메인과 독립적인 개인 블로그를 고민했다.
- 네이버 블로그와 SNS는 유입에 유리하지만 개인 사이트를 기록의 본진으로 보기로 했다.
- 이력서나 채용용 포트폴리오가 목적이 아니다.
- 개발, UI, 웹 접근성, 반응형 구현, 작업 방식과 AI 활용에 관한 배움과 시도를 기록한다.
- 운영 결과가 포트폴리오처럼 보일 수 있으나 목적이 아니라 결과다.
- Berkshire Hathaway 홈페이지의 장식 없는 링크 중심 정보 구조에 관심을 가졌다.
- 오래된 구현과 접근성 문제는 복제하지 않고 반응형과 접근성을 현대적으로 구현한다.
- 모바일과 PC에서 빠르고 편하게 읽는 사이트를 목표로 한다.
- AI는 초안을 돕지만 게시 전 직접 검토한다.

직장, 경력, 고객, 프로젝트, 수치, 성과와 자격은 추가하지 않았다.

## 사용자가 직접 검토할 문장과 판단

- “메모가 여러 서비스와 폴더에 흩어지면 … 맥락을 되짚기 어려웠다”가 실제 경험을 정확히 표현하는지
- 플랫폼은 유입과 대화의 통로, 개인 사이트는 원문과 축적의 중심이라는 구분에 동의하는지
- 해결하지 못한 문제와 판단을 바꾼 이유도 기록하겠다는 운영 원칙을 실제로 지킬 의향이 있는지
- 키보드 순서와 실제 기기 줄바꿈까지 확인한다는 표현이 현재 운영 계획과 맞는지
- AI가 “빠뜨린 항목을 찾는 도구”라는 표현과 게시 전 검토 범위가 실제 사용 방식과 맞는지
- `published: 2026-07-11`을 실제 공개일로 사용할지, 공개하는 날로 변경할지
