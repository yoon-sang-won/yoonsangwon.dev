# Pages CMS 초안 작성 흐름

검증일: 2026-07-11

## 이 프로젝트의 임시저장

이 프로젝트에서 임시저장은 글을 `draft: true`로 Pages CMS에서 저장해 GitHub commit을 만드는 것이다. 저장된 초안은 production 사이트의 홈페이지, collection 목록, Archive, RSS, sitemap과 공개 BlogPosting JSON-LD에서 제외된다.

브라우저에 입력만 한 내용은 임시저장이 아니다. Pages CMS의 저장 버튼을 눌러 GitHub에 반영된 내용만 다른 기기에서 다시 열 수 있다.

## 모바일에서 작성하고 PC에서 이어쓰기

1. 모바일에서 Pages CMS에 로그인한다.
2. Writing, Notes 또는 Lab에서 새 글을 만든다.
3. 영문 소문자와 하이픈으로 파일명을 확정한다.
4. `draft`가 켜져 있는지 확인한다.
5. 본문을 작성하고 저장한다.
6. GitHub commit 생성이 끝난 것을 확인한 뒤 모바일 편집을 종료한다.
7. PC에서 Pages CMS에 로그인한다.
8. 저장소와 글 목록을 새로고침한다.
9. 같은 초안을 열어 이어서 작성한다.
10. 다시 `draft: true` 상태로 저장한다.

## 저장과 자동 저장 제한

- 저장 버튼을 누른 내용만 GitHub에 남는다.
- 저장하지 않은 브라우저 입력은 복구를 보장하지 않는다.
- Pages CMS의 공식 자동 저장, 미저장 원고 복구와 기기 간 미저장 동기화는 확인되지 않았다.
- 브라우저의 localStorage나 sessionStorage에 편집 본문이 보관된다고 기대하지 않는다.
- 긴 글은 문단이나 절을 마칠 때마다 저장하는 것을 권장한다.
- 기기를 바꾸기 전 마지막 저장 commit을 확인한다.

### 공식 기능 확인 결과

확인 기준은 공식 문서와 공식 저장소의 현재 `main`, Pages CMS 2.1.8 (`6f4e860a35d934406580287e7042e5e111e207a1`)이다. 공식 저장소는 hosted 앱이 최신 버전을 사용한다고 안내하지만, 로그인 전 hosted 화면에서는 배포 commit 식별자를 노출하지 않는다. 따라서 아래 결과는 공식 최신 소스를 hosted 동작의 근거로 삼되, 화면에서 직접 실행했다고 주장하지 않는다.

| 확인 항목 | 공식 문서·소스에서 확인한 내용 | 이 프로젝트의 운영 판단 |
| --- | --- | --- |
| 편집 중 자동 저장 | 공식 문서는 편집 후 GitHub에 저장하는 흐름만 설명한다. 소스는 저장 버튼 또는 `Ctrl/Cmd+S`로 명시적 제출을 실행하며, 주기적 자동 제출은 확인되지 않았다. | 자동 저장 미지원으로 취급한다. |
| localStorage/sessionStorage의 미저장 입력 보관 | localStorage는 최근 방문 저장소 목록, sessionStorage는 작업 알림 상태에 사용된다. 편집 필드나 Markdown 본문을 저장하는 사용은 확인되지 않았다. | 미저장 원고 복구 수단으로 사용하지 않는다. |
| 브라우저 종료 후 복구 | 편집 본문을 영속 저장하는 근거가 없고 공식 복구 안내도 확인되지 않았다. | 종료 전 저장하지 않은 내용은 복구되지 않을 수 있다. |
| 저장되지 않은 변경의 페이지 이탈 경고 | 폼의 변경 여부 추적은 존재하지만 `beforeunload` 또는 동등한 이탈 차단 처리는 확인되지 않았다. | 경고가 나타난다고 기대하지 않는다. |
| 여러 기기 사이의 미저장 내용 동기화 | 공식 동작은 저장한 파일을 GitHub에 기록하는 방식이다. 미저장 브라우저 상태를 서버나 다른 기기에 동기화하는 기능은 확인되지 않았다. | GitHub commit으로 저장된 내용만 기기 간 이동한다. |

자동 저장이 반드시 필요하다면 Pages CMS의 현재 기본 기능 범위를 벗어난다. 이 프로젝트에 커스텀 자동 저장이나 별도 데이터베이스를 추가하지 않으며, 요구가 생기면 다른 편집 도구나 별도 운영 방식을 독립적으로 검토한다.

## 동시 편집 금지

모바일과 PC에서 같은 글을 동시에 열어 수정하지 않는다. 두 기기가 서로의 미저장 상태를 알지 못하고, 먼저 열어 둔 오래된 화면에서 나중에 저장하면 최신 commit의 내용을 덮거나 충돌을 일으킬 수 있다. 먼저 연 기기에서 저장하고 편집을 종료한 뒤, 다음 기기에서 목록을 새로고침해 최신 상태를 연다. Pages CMS는 실시간 공동 편집 시스템으로 사용하지 않는다.

## 다른 기기에서 최신 버전 열기

1. 이전 기기에서 저장 완료 알림을 확인한다.
2. GitHub에서 해당 Markdown 파일의 최신 commit과 변경 시각을 확인한다.
3. 이전 기기의 편집 화면을 닫는다.
4. 새 기기에서 Pages CMS의 같은 저장소와 branch를 연다.
5. Writing, Notes 또는 Lab 목록을 새로고침한다.
6. 같은 파일을 열고 마지막 문단이 있는지 확인한 뒤 편집한다.

목록이나 편집 화면이 이미 열려 있었다면 반드시 새로고침한다. 최신 내용이 보이지 않으면 저장소의 commit을 먼저 확인하고, 오래된 화면에서 저장하지 않는다.

## 공개 전환과 공개 중단

- 공개하려면 게시일과 내용을 확인하고 `draft`를 끈 뒤 저장한다.
- GitHub Actions 배포가 성공하면 사이트에 반영된다.
- 공개를 잠시 중단하려면 파일을 삭제하지 말고 `draft: true`로 되돌린다.
- 완전히 폐기할 글만 삭제한다.

## Git 기록에서 초안 복구

잘못 저장한 경우 GitHub에서 해당 Markdown 파일의 History를 열고 정상 commit의 내용을 복원한다. 삭제한 초안은 삭제 직전 commit에서 파일을 복구할 수 있다.

## 공개 저장소 주의

`draft: true`는 공개 사이트에서만 글을 숨긴다. GitHub 저장소가 공개되어 있으면 초안 Markdown 파일과 commit은 누구나 볼 수 있다.

초안에도 다음 내용을 기록하지 않는다.

- 회사 내부 정보와 회사 코드 원본
- 고객·동료의 개인정보
- 개인 연락처
- API key, token, password
- 비공개 계약 또는 관리 화면 정보
- 공개 전 유출되면 곤란한 자료

## 기기 변경 전 체크리스트

- `draft: true`인지 확인했다.
- 저장 버튼을 눌렀다.
- GitHub commit 생성이 끝났다.
- 현재 기기의 편집 화면을 닫았다.
- 새 기기에서 목록을 새로고침했다.
- 최신 내용을 확인한 뒤 편집을 시작했다.
