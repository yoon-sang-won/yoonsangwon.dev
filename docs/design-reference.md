# Berkshire Hathaway 디자인 레퍼런스

## 확인 정보

- 확인일: 2026-07-10
- 대상: Berkshire Hathaway 공식 홈페이지
- URL: https://www.berkshirehathaway.com/
- 확인 범위: 현재 홈페이지의 실제 콘텐츠 순서, 데스크톱·모바일 렌더링, 링크와 부가정보 표현, 문서 구조

이 문서는 해당 사이트의 상표나 외형을 복제하기 위한 자료가 아니다. 직접적인 정보 제공 태도와 링크 인덱스 구조를 `yoonsangwon.dev`에 맞게 재해석하기 위한 관찰 기록이다.

## 실제 페이지의 콘텐츠 순서

2026-07-10 현재 홈페이지는 다음 순서로 구성되어 있다.

1. 페이지 중앙의 회사명
2. 우편 주소
3. `Official Home Page` 문구
4. 전체 폭의 가는 구분선
5. 2열 표 안의 핵심 문서 링크 16개
6. 일부 링크 아래의 `Updated ...` 날짜
7. 전체 폭의 가는 구분선
8. GEICO 로고, 홍보 문구, 웹사이트와 전화번호
9. 전체 폭의 가는 구분선
10. 웹페이지 의견 접수와 답변 제한에 관한 안내문
11. 전체 폭의 가는 구분선
12. Legal Disclaimer 링크
13. 저작권 문구

회사 소개를 길게 설명하지 않고 정체성 표시 직후 핵심 문서로 이동하게 한다. 페이지의 대부분은 설명문이 아니라 링크다.

## 링크 목록의 역할

- 링크 목록이 홈페이지의 주 인터페이스이자 사이트맵 역할을 한다.
- 보고서, 공시, 회의, 편지, 지배구조 등 사이트의 핵심 영역으로 바로 이동시킨다.
- 각 링크는 목적지가 비교적 구체적이며 별도의 카드나 소개 화면을 거치지 않는다.
- 링크는 기본 브라우저에 가까운 밑줄 스타일을 사용한다.
- 방문한 링크는 보라색으로 구분되어 탐색 이력을 자연스럽게 드러낸다.
- 데스크톱에서는 링크를 2열로 나눠 많은 항목을 한 화면에서 훑게 한다.

`yoonsangwon.dev`에서는 이 역할을 Writing, Notes, Lab, Archive, About, Colophon과 최신 콘텐츠 링크가 담당한다.

## 구분선과 여백의 사용

- 얇은 수평선이 정체성, 핵심 링크, 광고, 안내문, 법적 정보의 큰 영역을 나눈다.
- 링크 항목 사이에는 별도 상자 없이 행 간격만 둔다.
- 바탕은 흰색이고 그림자, 색상 패널, 카드 배경이 없다.
- 중앙의 정체성 블록은 짧고 조밀하다.
- 넓은 화면에서 2열 목록이 전체 폭을 사용해 본문형 사이트보다 가로로 넓다.

가져올 것은 “상자 대신 구분선과 간격으로 영역을 나누는 방식”이다. 원본의 과도하게 넓은 2열 폭은 가져오지 않는다.

## 날짜와 부가정보 표시 방식

- 모든 링크에 날짜를 붙이지 않는다.
- 최근성이나 갱신 상태가 중요한 문서에만 링크 아래 작은 글자로 `Updated May 2, 2026`처럼 표시한다.
- 링크보다 날짜의 시각적 위계가 낮다.
- 날짜가 링크 행을 복잡하게 만들지 않고 보조 정보로 남는다.

`yoonsangwon.dev`에서는 모든 콘텐츠 항목에 게시일을 표시하고, 수정일은 게시일과 의미 있게 다를 때만 추가한다. 화면에는 짧은 ISO 형식 `YYYY-MM-DD`을 사용하고 마크업은 `<time datetime="...">`으로 제공한다.

## 안내문과 footer 구조

- 핵심 링크와 별개로 페이지 하단에 짧은 운영 안내문을 둔다.
- 안내문은 연락 가능성과 답변 제한을 솔직하게 설명한다.
- 마지막에는 법적 고지 링크와 저작권이 남는다.
- 별도의 마케팅 CTA나 뉴스레터 가입 영역은 없다.

`yoonsangwon.dev`에서는 다음으로 재해석한다.

- 블로그 운영 취지
- 오류·피드백을 보낼 경로
- 글은 개인적인 학습과 의견이며 소속 조직을 대변하지 않는다는 고지
- 비공개 업무 정보를 다루지 않는다는 고지
- 저작권과 Colophon 링크

## 데스크톱과 모바일 관찰

### 데스크톱

- 회사명과 주소는 중앙 정렬이다.
- 링크는 2열 표로 정렬된다.
- 각 행의 좌우 링크 길이가 달라도 같은 행을 공유한다.
- 긴 링크는 셀 안에서 줄바꿈한다.
- 목록 항목 사이의 넉넉한 세로 간격 덕분에 항목 구분은 가능하다.

### 모바일

- 가로 스크롤은 발생하지 않았지만 2열 표를 그대로 유지한다.
- 각 열이 지나치게 좁아져 긴 링크가 여러 줄로 잘게 끊어진다.
- 날짜와 링크의 밀도가 높아 빠르게 읽기 어렵다.
- 모바일 전용 재배치나 1열 전환이 없다.

`yoonsangwon.dev`는 표가 아닌 목록을 사용한다. 넓은 화면에서는 링크와 날짜를 같은 행에 놓을 수 있지만 좁은 화면에서는 날짜가 다음 줄로 자연스럽게 내려가야 한다.

## 개인 블로그에 가져올 요소

1. 사이트 주체를 첫 화면 상단에서 짧고 명확하게 표시한다.
2. 장문 소개보다 실제 콘텐츠 링크를 먼저 제공한다.
3. 홈페이지 자체를 콘텐츠 인덱스로 사용한다.
4. 링크 목록을 주 인터페이스로 사용한다.
5. 구분선과 간격으로 큰 정보 영역을 나눈다.
6. 날짜를 링크보다 낮은 위계의 부가정보로 표시한다.
7. 하단에 짧은 운영 안내와 고지를 둔다.
8. 저작권과 제작 정보를 명확히 남긴다.
9. 링크의 방문 상태를 구분한다.
10. 장식보다 기능과 문서 접근을 우선한다.

## 가져오지 않을 요소

- Berkshire Hathaway의 회사명, 로고, 주소, 문구
- 기업 보고서와 주주 문서 분류
- GEICO 광고와 외부 회사 홍보
- table 기반 레이아웃
- 모바일에서도 유지되는 2열 링크 구조
- 문단만으로 만든 정체성 영역
- 의미론적 landmark와 heading이 부족한 마크업
- 작은 글자와 좁은 모바일 링크 열
- 목적지가 모호하거나 문맥 의존적인 링크 문구
- 오래된 HTML 관행과 HTTP 외부 링크

## 현대적으로 보완할 부분

- `header`, `nav`, `main`, `section`, `footer`를 사용한다.
- 페이지마다 하나의 명확한 `h1`과 순차적인 heading 계층을 둔다.
- skip link와 강한 `:focus-visible` 스타일을 제공한다.
- 링크 목록은 `ul`과 `li`로 작성한다.
- 모바일에서는 1열로 쌓고 터치 영역과 행 간격을 확보한다.
- 긴 제목, URL, 코드가 컨테이너를 넘지 않게 한다.
- 시스템 글꼴과 상대 단위를 사용해 글자 확대를 지원한다.
- 링크는 색상과 밑줄로 식별하고 방문 상태도 충분한 대비를 유지한다.
- 게시일과 수정일은 `time` 요소로 제공한다.
- 고대비 모드와 reduced motion 환경에서 정보가 사라지지 않게 한다.
- 페이지별 title, description, canonical, RSS, sitemap을 제공한다.

## 적용 결론

핵심 레퍼런스는 파란 링크나 오래된 외형이 아니라 다음 흐름이다.

> 정체성 표시 → 구분선 → 핵심 링크 인덱스 → 갱신 정보 → 운영 안내 → 고지와 저작권

`yoonsangwon.dev`는 이 흐름을 유지하되, 링크 목록을 시맨틱하고 반응형인 1열 우선 구조로 다시 구현한다.
