# 콘텐츠 Markdown 템플릿

아래 템플릿은 출발점일 뿐이다. 필요 없는 소제목은 지우고 내용에 맞는 제목을 추가한다. `YYYY-MM-DD`, 제목, 설명과 파일명은 반드시 실제 값으로 바꾼다. 새 글은 항상 `draft: true`로 시작한다.

Pages CMS에서는 frontmatter 필드를 화면에서 입력하고 본문 부분만 복사해도 된다. 파일명이 공개 URL이 되므로 생성할 때 영문 소문자와 하이픈으로 확정한다.

## Writing

```md
---
title: "글 제목"
description: "다루는 문제와 글에서 확인할 수 있는 내용을 한두 문장으로 설명합니다."
published: YYYY-MM-DD
draft: true
---

글을 쓰게 된 문제나 질문을 짧게 설명합니다.

## 맥락

독자가 판단을 이해하는 데 필요한 배경만 적습니다.

## 핵심 내용

확인한 근거, 판단 과정과 배운 점을 내용에 맞게 나눕니다.

## 남은 점

아직 확인하지 못한 부분이나 다음에 살펴볼 내용을 적습니다.

## 참고

- [자료 이름](https://example.com)
```

## Notes

`description`과 소제목은 필요 없으면 삭제한다.

````md
---
title: "메모 제목"
description: "목록에서 별도 설명이 필요할 때만 작성합니다."
published: YYYY-MM-DD
draft: true
---

무엇을 알게 되었는지 또는 어떤 문제를 해결했는지 바로 적습니다.

```text
필요한 명령어나 짧은 코드
```

- 확인한 링크: [자료 이름](https://example.com)
- 남은 질문: 추후 확인할 내용
````

## Lab

```md
---
title: "실험 제목"
description: "무엇을 어떤 조건에서 확인한 실험인지 한두 문장으로 설명합니다."
published: YYYY-MM-DD
draft: true
---

## 질문

이번 실험에서 확인하려는 내용을 적습니다.

## 조건

- 환경 또는 버전:
- 비교 기준:

## 시도

직접 실행한 절차와 필요한 코드를 적습니다.

## 결과

관찰한 결과를 예상이나 해석과 구분해 적습니다.

## 한계와 다음 확인

이번 결과가 적용되지 않는 조건과 다음 실험을 적습니다.
```

분류 기준은 [콘텐츠 편집 원칙](editorial-guide.md#writing-notes-lab-구분), 공개 전에는 [발행 체크리스트](publishing-checklist.md)를 확인한다.
